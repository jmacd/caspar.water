# Watershop Selfmon Metrics

This Watertown instance ingests host journald, Caddy access logs, and
its own per-tick performance metrics every 60 seconds.  The charts
below show how Watertown's read/list/memory cost grows with commit
count -- the data drives compaction, checkpointing, and vacuum work.

Generated by **sitegen** running on the same pond it monitors -- the
sitegen invocation cost itself is included in the peak-RSS and
runtime series ("self-referential observability").
